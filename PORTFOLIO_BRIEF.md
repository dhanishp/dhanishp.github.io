# Portfolio Website Brief

## Owner
Dhanish Parimalakumar

## Goal
Create a modern, personal portfolio website that is aesthetically strong and professional enough for data science recruiting. It should feel like a polished personal site, not just an online resume.

## Audience
- Data science recruiters
- Hiring managers
- technical interviewers
- professors and mentors
- professional peers

## Site goals
- Make a strong first impression within 5 seconds
- Showcase high-impact work clearly
- Present both technical depth and business relevance
- Feel personal, modern, and credible
- Be easy to update
- Deploy on GitHub Pages

## Core design decisions
- One-page scrolling layout
- Light/dark mode toggle
- Clean modern design
- Personal portfolio vibe, but still professional
- Subtle but noticeable animations when appropriate
- Headshot can appear in About section, but should not dominate the homepage
- Focus on clarity, polish, and strong visual hierarchy

## Interaction model
- Main website is a single-page scrolling experience
- Featured work and selected experience items can open deeper detail views
- Prefer modal or slide-over detail panels first
- Detail views should feel smooth and premium, not cluttered

## Primary content focus
This site should be optimized primarily for data science recruiting.

## Top featured work
1. GoFi - Loan Capture Modeling
2. VIEWS Lab - Conflict Fatality Forecasting
3. Verizon - Reporting Automation / API Architecture

## Secondary work to include
- Bitcoin Price Prediction using LSTM
- Agentic AI Whiteboarding & Product Management Tool
- Automated Mortgage Lending Application

## Desired sections
- Hero
- About
- Selected Work
- Experience
- Leadership / Activities
- Skills / Tech Stack
- Contact

## Hero direction
The hero should immediately communicate:
- who I am
- what I do
- the kind of work I am proud of

Avoid generic phrases like "student | developer | researcher."
Prefer sharper, more differentiated copy.

## Visual direction
- Premium typography
- Strong spacing and layout
- Subtle gradients / ambient effects allowed
- Clean cards
- Good contrast in both light and dark mode
- Avoid overdesigned dashboards or noisy futuristic visuals
- Avoid generic portfolio template feel

## Technical preferences
- Vite
- React
- Tailwind
- Framer Motion for subtle motion
- Responsive design
- Easy GitHub Pages deployment

## Content principles
- Emphasize metrics and outcomes
- Make case studies scannable
- Use concise, strong wording
- Show both technical and business skills
- Avoid filler text
- Avoid long dense paragraphs

## Constraints
- Must work well on GitHub Pages
- Must be maintainable
- Must look polished on desktop and mobile
- Must not feel cluttered
